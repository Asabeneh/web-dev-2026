## How the Web Works â€“ Restaurant Analogy
Imagine ordering food at a restaurant:

- **You (the client/browser)**: Look at the menu (request a page), place an order (send HTTP request)
- **Kitchen & staff (the server)**: Prepare the food (process request, fetch/store data), send it back
- **HTTP protocol**: The "language" used for communication between client and server

Most websites involve this client-server model. Understanding this big picture helps before learning specific technologies.

## Key Concepts Covered on Day 1

### 1. Command Line Basics
Use the terminal (Git Bash) to:
- Create folders and files
- Navigate directories
- Create basic HTML files

Practice common commands (covered in the recommended Git/GitHub materials).

### 2. HTML Basics
HTML structures web page content.

Basic document structure:
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Page</title>
</head>
<body>
    <!-- This is a comment -->
    <h1>Main Heading (H1)</h1>
    <h2>Subheading (H2)</h2>
    <p>This is a paragraph. Use Lorem Ipsum for placeholder text when needed.</p>
</body>
</html>
```

- Headings: `<h1>` to `<h6>` (semantic importance decreases from h1 â†’ h6)
- Paragraphs: `<p>`
- Comments: `<!-- comment here -->`
- Meta tags: Important for SEO and mobile responsiveness

### 3. CSS Basics (Introduction)
- Controls appearance (colors, spacing, layout)
- Uses the **box model**: Every element is a box with content, padding, border, and margin

More CSS and styling will be covered in upcoming weeks.

### 4. Semantic HTML5
Use meaningful tags (e.g., `<section>`, `<header>`, `<footer>`, `<nav>`, `<article>`) instead of generic `<div>` for better accessibility, SEO, and maintainability.

### 5. Git, GitHub & Deployment
GitHub provides:
- Version control (track changes)
- Collaboration
- Free hosting for static websites

Basic workflow demonstrated:
1. Create a repository on GitHub
2. Write code locally (e.g., in VS Code)
3. Commit changes
4. Push to GitHub
5. Deploy (GitHub Pages turns repositories into live websites)
